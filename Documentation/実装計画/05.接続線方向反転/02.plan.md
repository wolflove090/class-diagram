# 接続線方向反転 実装計画書

## 実装方針

方向反転は表示専用のフラグを増やさず、関係が持つ両端のデータをモデル層で交換する。`DiagramModel.reverseRelationship(state, relationshipId)` を追加し、対象関係の `sourceClassId` と `targetClassId`、`sourceMultiplicity` と `targetMultiplicity` を同時に交換する。

インスペクターの関係編集フォームに `type="button"` の「方向を反転」ボタンを置き、クリックイベントをコントローラーへ渡す。`DiagramController` はモデルメソッドで状態を更新し、既存の `updateState` を通じて SVG 再描画、インスペクター再描画、自動保存を行う。

SVG レンダラーと Mermaid シリアライザーはいずれも既に `sourceClassId`／`targetClassId` と各端の多重度を基準に表示・出力しているため、反転操作のための変更は不要とする。この方針により関係種別にかかわらず矢印・ひし形の位置と Mermaid の方向が一貫して反転する。

## 変更対象ファイル一覧

- `src/models/diagramModel.js`
  - `reverseRelationship` を追加し、関係の両端と両端多重度を原子的に交換する。
- `src/views/inspectorView.js`
  - 関係編集フォームに方向反転ボタンと `onRelationshipReverse` コールバックを追加する。
- `src/controllers/diagramController.js`
  - インスペクターの方向反転イベントを受け、モデル更新を `updateState` に渡す。
- `tests/mermaidParser.test.js`
  - モデルでの反転、Mermaid 出力、再インポート後の保持を検証するテストを追加する。

変更不要:

- `src/views/svgRenderer.js`: 接続元・接続先から経路を描画し、種別ごとにマーカーを配置済みであるため、反転後の状態をそのまま描画できる。
- `src/services/mermaidSerializer.js`: 接続元・接続先と各端多重度を出力済みであるため、反転後の状態をそのまま出力できる。
- `src/services/mermaidParser.js`: Mermaid の方向を既存形式でモデルへ正規化済みであり、変更不要。

`DiagramModel` は関係の意味を保ったまま両端のデータを交換する唯一の責務を持つ。`InspectorView` は反転要求の発行のみ、`DiagramController` は既存の状態更新経路への接続のみを担う。描画とエクスポートは更新済みの状態を読むため、個別の反転処理を追加しない。

## 処理フロー

### 関係を反転する

```text
利用者
  |
  | 「方向を反転」クリック
  v
InspectorView.renderRelationship()
  |
  | onRelationshipReverse(relationshipId)
  v
DiagramController.reverseRelationship()
  |
  | DiagramModel.reverseRelationship(state, relationshipId)
  v
新しい DiagramState
  |
  v
DiagramController.updateState()
  |-- SvgRenderer.render(state)       : マーカー・線の向きを更新
  |-- InspectorView.render(state)     : 接続元/先・多重度を更新表示
  `-- StorageService.save(state)      : 反転後の状態を保存
```

モデルで両端をまとめて交換してから既存の状態更新経路へ流す。これにより画面、保存、エクスポートの状態差異を生じさせない。

### Mermaid エクスポートと再インポート

```text
反転済み DiagramState
  |
  v
MermaidSerializer.serialize()
  |
  | source -> target と両端多重度を出力
  v
Mermaid テキスト
  |
  v
MermaidParser.parse()
  |
  v
反転済みの接続元・接続先・多重度を持つ DiagramState
```

シリアライザーとパーサーは既存の方向表現を使う。反転専用のメタデータを追加しないため、Mermaid 互換性を維持する。

## リスクと対策

- 多重度を交換し忘れると関係の意味が壊れる。
  - クラス ID と多重度を同じモデルメソッドで交換し、テストで両方を確認する。
- 継承・集約などはマーカー位置の基準が種別により異なる。
  - レンダラーの既存仕様が接続元・接続先に従うことを前提に、反転後 Mermaid 出力とモデル値をテストする。必要なら手動確認で全種別を選択する。
- 自己関連ではクラス ID の交換が視覚的に分かりにくい。
  - 多重度を交換するテストを追加し、操作の一貫性を保つ。

## 検証方針

1. `node --test tests/mermaidParser.test.js` を実行する。
2. 通常関係を反転し、接続元・接続先、両端多重度、ラベル、種別、ID を確認する。
3. 継承、実装、依存、関連、集約、コンポジションについて、SVG のマーカーが新しい方向に描画されることをブラウザで確認する。
4. Mermaid エクスポート後のテキスト、およびそのテキストを再インポートした状態を確認する。
5. 反転後に再読み込みし、保存済みの状態が復元されることを確認する。

## コードスニペット

### `src/models/diagramModel.js`

```javascript
  updateRelationship(state, relationshipId, patch) {
    return {
      ...state,
      relationships: state.relationships.map((relationship) => (
        relationship.id === relationshipId ? { ...relationship, ...patch } : relationship
      ))
    };
  }

+  reverseRelationship(state, relationshipId) {
+    return {
+      ...state,
+      relationships: state.relationships.map((relationship) => (
+        relationship.id !== relationshipId ? relationship : {
+          ...relationship,
+          sourceClassId: relationship.targetClassId,
+          targetClassId: relationship.sourceClassId,
+          sourceMultiplicity: relationship.targetMultiplicity,
+          targetMultiplicity: relationship.sourceMultiplicity
+        }
+      ))
+    };
+  }
```

+test("reverses relationship endpoints and multiplicities without changing its identity", () => {
