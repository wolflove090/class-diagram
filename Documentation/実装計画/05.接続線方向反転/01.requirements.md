# 接続線方向反転 要件書

## 背景と目的

クラス図上の接続線は、接続元・接続先のクラス、矢印／ひし形の向き、多重度、Mermaid 出力の方向を一つの関係データとして扱っている。接続後に関係の向きを修正したい場合、利用者は接続元・接続先と両端の多重度を個別に変更しなければならない。

選択中の接続線をワンアクションで反転し、関係の意味と表示を一貫して入れ替えられるようにする。

## スコープ内

- 選択中の関係に対する「接続線を逆方向にする」操作の追加。
- 関係の接続元と接続先の入れ替え。
- 接続元多重度と接続先多重度の入れ替え。
- 再描画、自動保存、Mermaid エクスポートへの反映。
- モデル操作と Mermaid 出力の自動テスト追加。

## スコープ外

- 関係種別そのものの変換（例: 継承を実装へ変更）。
- 片側だけの矢印・マーカーの独立した反転設定。
- Mermaid インポート構文の拡張。
- 複数の関係をまとめて反転する操作。
- キーボードショートカットの追加。

## 機能要件

1. 関係を選択したときに表示するインスペクターに、方向反転ボタンを表示する。
2. ボタン操作で、対象関係の `sourceClassId` と `targetClassId` を入れ替える。
3. 同じ操作で、`sourceMultiplicity` と `targetMultiplicity` を入れ替える。
4. `type`、`label`、関係 ID は変更しない。
5. 更新後、SVG の接続線は新しい接続元・接続先に基づいて描画される。継承・実装・集約・コンポジションではマーカーの位置も反対側になる。
6. 更新後、選択状態は対象関係のまま維持する。
7. 更新後の状態は既存の保存処理により LocalStorage へ保存され、Mermaid エクスポートでは反転後の方向・多重度で出力される。
8. 接続元と接続先が同一の自己関連でも操作は可能とする。この場合、クラス ID は変化しないが、両端の多重度は入れ替える。

## 非機能要件

- 既存の関係作成、関係編集、削除、SVG 描画、Mermaid インポート／エクスポートを壊さない。
- 保存済みの既存図データは形式変更なしで読み込める。
- 操作は既存のモデル更新 → 再描画 → 自動保存の経路を使用し、DOM を直接書き換えない。
- 操作後にブラウザを再読み込みしても反転後の状態を復元できる。

## 受け入れ条件

- 関連 `A --> B` を反転すると、図と Mermaid 出力が `B --> A` 相当になる。
- 継承 `Parent <|-- Child` を反転すると、図の中空三角と Mermaid 出力が子側から親側へ向く状態（`Child <|-- Parent` 相当）になる。
- 接続元多重度 `1`、接続先多重度 `0..*` の関係を反転すると、反転後は接続元が `0..*`、接続先が `1` になる。
- ラベル、関係種別、関係 ID は反転前後で同一である。
- 反転後の関係をエクスポートし、再インポートしても、接続元・接続先・多重度が反転後の状態で保持される。
- 関係が未選択の場合は方向反転操作を表示しない。
