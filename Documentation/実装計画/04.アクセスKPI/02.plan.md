# アクセスKPI集計 実装計画

## 目的

GitHub Pagesで公開する設計メーカーの表示回数を、Googleスプレッドシートへ日別に集計する。

## 方針

- ブラウザごとに生成した匿名IDを`localStorage`へ保存する。
- クライアントはJSTの当日について未送信の場合だけ、GAS Webアプリへ送信する。
- GASはJSTの日付をサーバー側で決定し、匿名IDのSHA-256ハッシュを重複判定に使う。
- `DailyKPI`シートには日付と表示回数だけを保存する。`VisitorLog`シートは重複判定用の内部データで、初回作成時に非表示にする。

## 制約

- ログイン情報を使わないため、識別単位は「同一ブラウザ・同一ストレージ」である。ブラウザデータ削除・別ブラウザ・別端末は別ユーザーとして数える。
- 公開されたGAS Webアプリは不正なリクエストを完全には防げない。KPIは利用傾向の把握を目的とする。
