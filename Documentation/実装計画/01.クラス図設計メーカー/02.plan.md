# クラス図設計メーカー 実装計画書

## 実装方針

プレーンな HTML/CSS/JavaScript と SVG で、ローカルブラウザ上で動作するクラス図エディタを実装する。ビルド工程やローカル静的サーバーを不要にするため、JavaScript は通常の `script` として依存順に読み込み、`index.html` をファイルとして直接開いて動作させる。

クラス図の編集状態は、描画用 DOM から切り離した内部データモデルで管理する。SVG はそのモデルを表示するビューとして扱い、ユーザー操作はコントローラーを通してモデルへ反映する。Mermaid インポート、Mermaid エクスポート、LocalStorage 自動保存も同じ内部データモデルを介して実行し、変換結果の差異を抑える。

関係は接続線方式で表現する。関係ノードは作成せず、クラスノード同士を SVG の線で直接接続する。関係種別は線種、矢印、ひし形マーカー、ラベルで表現する。

## データモデル

内部データは以下の形で保持する。

```javascript
{
  version: 1,
  classes: [
    {
      id: "class_1",
      name: "User",
      kind: "class",
      stereotype: "",
      visibility: "public",
      isAbstract: false,
      position: { x: 120, y: 80 },
      size: { width: 220, height: 140 },
      properties: [
        {
          id: "prop_1",
          name: "id",
          type: "string",
          visibility: "private",
          isStatic: false,
          isReadonly: true,
          isAbstract: false,
          defaultValue: "",
          description: ""
        }
      ],
      methods: [
        {
          id: "method_1",
          name: "getName",
          returnType: "string",
          visibility: "public",
          isStatic: false,
          isAbstract: false,
          isAsync: false,
          parameters: [],
          description: ""
        }
      ]
    }
  ],
  relationships: [
    {
      id: "rel_1",
      sourceClassId: "class_1",
      targetClassId: "class_2",
      type: "dependency",
      label: "",
      sourceMultiplicity: "",
      targetMultiplicity: ""
    }
  ],
  viewport: {
    x: 0,
    y: 0,
    scale: 1
  },
  selection: {
    type: "class",
    id: "class_1"
  }
}
```

### 接続線作成

```text
User starts connection from class handle
  |
  v

  |
  v

  |
  v

  |
  v

  |
  v
SvgRenderer.render(state)
```

接続作成時は、接続元と接続先のクラスが存在すること、同一 ID の不正接続でないことを検証する。自己参照は UML として必要になる可能性があるため、初期実装では許可するが、視認性を確保するためループ線として描画する。

## Mermaid 変換方針

### エクスポート

- 先頭に `classDiagram` を出力する。
- クラスは `class ClassName { ... }` 形式で出力する。
- 可視性は Mermaid の `+`, `#`, `-`, `~` に変換する。
- static は `$`、abstract は `*` を使う。
- プロパティは `visibility name type` を基本形にする。
- メソッドは `visibility methodName(param: Type) ReturnType` を基本形にする。
- 関係は Mermaid の代表的な関係記法へ変換する。

```text
inheritance     : Parent <|-- Child
implementation  : Interface <|.. Class
dependency      : Source ..> Target
association     : Source --> Target
aggregation     : Whole o-- Part
composition     : Whole *-- Part
```

### インポート

- `classDiagram` 行を検出する。
- `class ClassName { ... }` ブロックを解析する。
- `ClassName : +field string` のような単行メンバー定義も解析対象にする。
- 関係行を解析し、対応する `Relationship.type` に変換する。
- 未対応行は無視する。ただし、クラス、メンバー、関係のいずれも読み取れなかった場合はエラーにする。
- 日本語名は内部データとして保持する。
- スペースや Mermaid と衝突しやすい記号を含む名称は、エクスポート時に警告または安全な識別子へ正規化する。

## SVG 描画方針

- SVG 内に `viewportGroup` を作り、パン・ズームはこのグループの `transform` に反映する。
- クラスノードは `g` 要素として描画し、内部に `rect`、`text`、区切り線を配置する。
- 接続線はクラスノードより背面に描画する。
- 関係種別ごとの marker を `defs` に定義する。
- 選択中のクラスノードや接続線には専用 CSS クラスを付与する。
- 接続線のクリック判定用に、透明で太い `path` を見た目の線の上に重ねる。

## リスクと対策

- リスク: プレーン JavaScript で状態管理が散らかる。
  - 対策: `DiagramModel` と `AppController` に更新経路を集約し、ビューから直接状態を書き換えない。
- リスク: SVG の接続線クリック判定が難しい。
  - 対策: 見た目用 path と操作用 hit path を分け、hit path は透明かつ太めにする。
- リスク: Mermaid 構文の全対応が膨らむ。
  - 対策: 初期実装は `classDiagram` のクラス、メンバー、代表的な関係に限定し、未対応行は警告または無視する。
- リスク: 日本語や記号を含む名称の Mermaid 出力が壊れる。
  - 対策: 内部データでは保持し、出力時に安全性を検査する。危険な名称は警告し、必要に応じて正規化する。
- リスク: LocalStorage の容量や破損で復元できない。
  - 対策: 保存時に `try/catch` を置き、読み込み時にバージョンと最低限の構造を検証する。
- リスク: 大きな図で再描画が重くなる。
  - 対策: 初期実装では全体再描画でシンプルに作り、性能問題が出た場合だけ差分更新へ拡張する。

## 検証方針

- ブラウザで `index.html` を開き、初期画面が表示されることを確認する。
- クラス追加、クラス選択、名称編集、プロパティ追加、メソッド追加が反映されることを確認する。
- 2つのクラスを接続し、関係種別を変えると線種と矢印が変わることを確認する。
- 接続線を選択して削除できることを確認する。
- クラス削除時に、そのクラスへ接続していた関係も削除されることを確認する。
- ノードドラッグで位置が変わり、接続線も追従することを確認する。
- パン、ズーム、表示位置リセットが動作することを確認する。
- Mermaid エクスポート結果に `classDiagram`、クラス、メンバー、関係が含まれることを確認する。
- Mermaid インポートでクラス、メンバー、関係が表示されることを確認する。
- 読み取り不能な Mermaid 入力でエラー表示され、既存図が壊れないことを確認する。
- ブラウザ再読み込み後に LocalStorage から復元されることを確認する。
- 日本語のクラス名、プロパティ名、メソッド名が表示、保存、復元されることを確認する。

### `src/models/diagramModel.js`

```javascript
+class DiagramModel {
+  createInitialState() {
+    return {
+      version: 1,
+      classes: [],
+      relationships: [],
+      viewport: { x: 0, y: 0, scale: 1 },
+      selection: null
+    };
+  }
+
+  addClass(state, input) {
+    const classNode = createClassNode(input);
+    return {
+      ...state,
+      classes: [...state.classes, classNode],
+      selection: { type: "class", id: classNode.id }
+    };
+  }
+
+  removeClass(state, classId) {
+    return {
+      ...state,
+      classes: state.classes.filter((item) => item.id !== classId),
+      relationships: state.relationships.filter((item) => (
+        item.sourceClassId !== classId && item.targetClassId !== classId
+      )),
+      selection: null
+    };
```

### `src/services/mermaidParser.js`

```javascript
+class MermaidParser {
+  parse(text) {
+    const lines = normalizeLines(text);
+    const classes = [];
+    const relationships = [];
+
+    for (const line of lines) {
+      if (isClassBlockStart(line)) {
+        classes.push(this.parseClassBlock(lines));
+      } else if (isRelationshipLine(line)) {
+        relationships.push(this.parseRelationship(line));
+      } else if (isSingleLineMember(line)) {
+        applyMemberLine(classes, line);
+      }
+    }
+
+    if (classes.length === 0 && relationships.length === 0) {
+      throw new Error("Mermaidからクラス図情報を読み取れませんでした。");
+    }
+
+    return { version: 1, classes, relationships, viewport: { x: 0, y: 0, scale: 1 }, selection: null };
+  }
+}
```
