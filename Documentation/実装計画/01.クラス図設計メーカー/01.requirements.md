# クラス図設計メーカー 要件書

## 背景と目的

ローカルブラウザ上で動作する、クラス図の作成・編集・Mermaid記法との相互変換が可能な設計支援ツールを作成する。

ユーザーはクラス、プロパティ、メソッド、クラス間の関係を視覚的に編集し、作成したクラス図を Mermaid の `classDiagram` 記法としてエクスポートできる。また、既存の Mermaid 記法で書かれたクラス図をインポートし、ビュワー上で再編集できる。

## スコープ内

- ローカルブラウザで動作するクラス図作成ツールを提供する。
- プレーンな HTML/CSS/JavaScript で実装する。
- 図の描画方式は SVG とする。
- クラス図を新規作成できる。
- クラスをノードとしてビュワー上に配置できる。
- クラス同士を接続線方式で接続できる。
- 接続済みの接続線を削除できる。
- クラス図全体をビュワー内で自由に移動できる。
- クラスごとにプロパティを設定できる。
- クラスごとにメソッドを設定できる。
- 一般的な UML クラス図で表現される主要情報を設定できる。
- 作成したクラス図全体を Mermaid 記法でエクスポートできる。
- Mermaid 記法で記述されたクラス図をインポートできる。
- インポートしたクラス図をビュワー上で編集できる。
- 編集状態を LocalStorage に自動保存できる。
- ファイルとしての入出力は Mermaid 形式を対象とする。

## スコープ外

- サーバーへの保存、共有、共同編集。
- ユーザー認証。
- 外部データベースとの連携。
- 複数ファイル形式へのエクスポート。ただし Mermaid 記法のテキスト出力はスコープ内とする。
- JSON ファイルの保存・読み込み。
- UML の全図種への対応。対象はクラス図のみとする。
- Mermaid の全構文への完全対応。対象は `classDiagram` のうち、本ツールの編集モデルで扱える範囲とする。
- React、Vue、Svelte などのフロントエンドフレームワークの利用。
- React Flow などのグラフ編集ライブラリの利用。

## 機能要件

### 1. アプリケーション実行

- ツールはローカルブラウザで実行できること。
- ツールはプレーンな HTML/CSS/JavaScript で構成すること。
- ビルド工程なし、または最小限の静的ファイル配信で起動できること。
- 初期表示時にクラス図編集用のビュワーを表示すること。
- 画面上にクラス追加、関係追加、削除、インポート、エクスポートの操作導線を用意すること。

### 2. クラスノード作成

- ユーザーは新しいクラスノードを作成できること。
- クラスノードには少なくともクラス名を設定できること。
- クラスノードにはステレオタイプを設定できること。
- クラスノードには可視性を設定できること。
- クラスノードには抽象クラス、インターフェース、列挙型などの種別を設定できること。
- クラスノードには任意の位置情報を保持できること。
- 作成済みクラスノードは選択、編集、削除できること。

### 3. プロパティ編集

- クラスノードにはプロパティを複数追加できること。
- プロパティには以下の情報を設定できること。
  - 名前
  - 型
  - 可視性
  - static
  - readonly
  - abstract
  - 初期値
  - 説明
- プロパティは追加、編集、削除、並び替えができること。

### 4. メソッド編集

- クラスノードにはメソッドを複数追加できること。
- メソッドには以下の情報を設定できること。
  - 名前
  - 戻り値の型
  - 可視性
  - static
  - abstract
  - async
  - パラメータ
  - 説明
- パラメータには以下の情報を設定できること。
  - 名前
  - 型
  - 既定値
  - 任意指定
- メソッドは追加、編集、削除、並び替えができること。

### 5. クラス間関係の作成

- ユーザーは2つのクラスノード間に関係を作成できること。
- 関係は接続線方式で表現すること。
- 関係は独立した関係ノードとしては扱わず、クラスノード間を直接結ぶ接続線として扱うこと。
- 接続線として扱うことで、UML クラス図として自然な見た目にし、継承や依存などの関係種別を線種と矢印で表現すること。
- 関係には以下の種類を設定できること。
  - 継承
  - 実装
  - 依存
  - 関連
  - 集約
  - コンポジション
- 関係には向きを設定できること。
- 関係にはラベルを設定できること。
- 関係には多重度を設定できること。
- 関係は選択、編集、削除できること。
- 接続元または接続先のクラスノード削除時は、関連する関係も削除されること。

### 6. ビュワー操作

- ユーザーはビュワー内でクラス図全体を移動できること。
- ユーザーはクラスノードをドラッグして移動できること。
- クラスノードの移動に合わせて、接続線の始点・終点も更新されること。
- ズームイン、ズームアウト、表示位置リセットができること。
- 大きなクラス図でも表示領域から要素を探せる操作性を確保すること。

### 7. Mermaid エクスポート

- 現在のクラス図全体を Mermaid の `classDiagram` 記法としてエクスポートできること。
- エクスポート対象は Mermaid の `classDiagram` を主対象とすること。
- `classDiagram-v2` 固有の記法は初期実装の主対象外とすること。
- クラス名、クラス種別、ステレオタイプ、プロパティ、メソッド、関係を出力できること。
- Mermaid で直接表現できない内部情報は出力対象外として扱うこと。
- エクスポート結果をユーザーがコピーできること。
- エクスポートするファイル形式は Mermaid 形式を対象とし、JSON ファイルのエクスポートは行わないこと。

### 8. Mermaid インポート

- ユーザーは Mermaid の `classDiagram` 記法を入力し、クラス図として取り込めること。
- インポート対象は Mermaid の `classDiagram` を主対象とすること。
- `classDiagram-v2` 固有の記法、クリックイベント、namespace などは初期実装の主対象外とすること。
- インポート時にクラス、プロパティ、メソッド、関係を解析できること。
- Mermaid 記法に構文エラーがある場合は、エラー箇所または原因をユーザーに表示すること。
- インポートしたクラス図は編集可能な内部データ構造に変換されること。
- Mermaid 記法から位置情報を取得できないため、インポート時は自動初期配置を適用すること。
- 自動初期配置では、取り込んだクラスノードが重なりにくいように一定間隔のグリッドまたは簡易レイヤー配置を行うこと。
- Mermaid から読み取れない内部情報は破棄してよいこと。
- 入力内容からクラス図として有効な情報を全く読み取れない場合は、その旨をユーザーに表示すること。
- 日本語のクラス名、プロパティ名、メソッド名を扱えること。
- 記号とスペースを含む名称は、Mermaid 記法として安全に扱える範囲に正規化、または入力時に警告すること。

### 9. 自動保存

- 編集中のクラス図は LocalStorage に自動保存されること。
- ブラウザを再読み込みした場合、最後に編集していたクラス図を復元できること。
- ユーザーが明示的に現在の編集状態をクリアできること。
- LocalStorage に保存するデータは、ツール内部のデータモデルを JSON 化した形式とすること。
- LocalStorage は内部自動保存専用とし、ユーザー向けの JSON ファイル保存・読み込み機能は提供しないこと。

## 非機能要件

- ローカル環境で完結して動作すること。
- プレーンな HTML/CSS/JavaScript と SVG を用いて実装し、フレームワーク依存を持たないこと。
- 主要操作はブラウザ上で即時に反映されること。
- クラス数が50、関係数が100程度の図でも実用的に操作できること。
- 編集中の状態は一貫性を保ち、不正な接続や存在しないクラスへの関係を保存しないこと。
- インポート、エクスポート処理は同一データモデルを介して実行し、変換結果の差異を抑えること。
- UI はクラス図編集を主画面とし、初期表示から主要操作にすぐアクセスできること。
- データモデル、Mermaid 変換処理、描画処理、UI 操作処理は責務を分離すること。
- LocalStorage の保存失敗時は、編集操作自体を妨げず、ユーザーに保存できなかったことを通知すること。

## 受け入れ条件

- ローカルブラウザでツールを起動できる。
- ユーザーが新規クラスノードを作成できる。
- クラスノードにプロパティとメソッドを追加できる。
- 2つのクラスノードを継承、依存、関連などの関係で接続できる。
- 作成済みの関係を削除できる。
- クラスノード削除時に関連する関係も削除される。
- ビュワー内でクラス図全体を移動できる。
- クラスノードを個別に移動できる。
- 作成したクラス図を Mermaid `classDiagram` 記法としてエクスポートできる。
- Mermaid `classDiagram` 記法をインポートして、編集可能なクラス図として表示できる。
- インポートした内容を再エクスポートしたとき、クラス、プロパティ、メソッド、関係の主要情報が保持される。
- 不正な Mermaid 入力に対して、ツールがクラッシュせずエラーを表示する。
- Mermaid から読み取れるクラス図情報が全くない入力に対して、読み取り不能であることを表示する。
- 編集状態が LocalStorage に自動保存され、再読み込み後に復元される。
- 日本語のクラス名、プロパティ名、メソッド名を含むクラス図を作成、表示、保存できる。

## 確定済みの設計判断

- Mermaid の対応範囲は `classDiagram` を主対象として確定する。`classDiagram-v2` 固有機能、クリックイベント、namespace は初期実装では対象外とする。
- 関係は接続線として扱う方針で確定する。
  - 関係ノード方式は、関係そのものを独立した箱として配置する方式であり、関係に多くの属性を持たせる場合には便利だが、一般的な UML クラス図の見た目から離れやすい。
  - 接続線方式は、クラス間を線で直接結ぶ方式であり、継承、依存、集約などを線種と矢印で表現できるため、今回のクラス図エディタに適している。
- 自動レイアウトは、Mermaid インポート時に位置情報がないクラスを初期配置する処理として扱う。高度な最適配置ではなく、重なりを避ける簡易配置を初期実装の対象とする。

## 未確定事項・確認事項

- 記号とスペースを含む名称の詳細な扱いは実装計画で決める。方針は、内部データでは保持し、Mermaid 出力時に安全な表現へ正規化または警告する。
- JSON ファイルとしての保存・読み込みを初期実装に含めるかは未確定。LocalStorage 自動保存はスコープ内として確定済み。
- JSON ファイルとしての保存・読み込みは初期実装では不要とする。ユーザー向けの入出力は Mermaid 形式で行う。
