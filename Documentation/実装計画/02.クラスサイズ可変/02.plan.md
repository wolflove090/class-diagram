# クラスサイズ可変 実装計画書

## 実装方針

既存の `classNode.size` を正式な保存対象として使い、SVG 上の選択済みクラスにリサイズハンドルを追加する。リサイズ操作は `SvgRenderer` の既存ドラッグ状態 `this.drag` に `resize` 種別を追加して扱う。リサイズ中は `DiagramController` にサイズ変更イベントを渡し、`DiagramModel.updateClass()` で対象クラスの `size` を更新する。

表示サイズは、保存済みサイズをそのまま使うのではなく、内容に必要な最小高さを下回らないように `SvgRenderer.getClassRenderSize(classNode)` に集約する。これにより、クラス本体、接続線の端点、範囲選択判定が同じ幅・高さを参照し、リサイズ後の線の追従や選択判定のずれを防ぐ。

リサイズハンドルは初期実装では右端、下端、右下角の3種類に限定する。左端・上端からのリサイズは、位置変更も同時に必要になり既存のクラス移動との競合が増えるため、今回の範囲外とする。

## 変更対象ファイル一覧

- `src/views/svgRenderer.js`
  - リサイズハンドル描画、リサイズ用 pointerdown、ドラッグ中のサイズ計算、リサイズ終了通知を追加する。
  - 表示サイズ計算を `getClassRenderSize()` に集約し、クラス本体、接続線、範囲選択で共有する。
- `src/controllers/diagramController.js`
  - `SvgRenderer` からの `onClassResize` と `onClassResizeEnd` を受け取り、モデル更新と保存を行う。
- `src/models/diagramModel.js`
  - `size` の正規化で最小サイズを保証する。
  - `createClass()` の既定サイズを最小サイズ以上に揃える。
- `src/styles.css`
  - リサイズハンドルの見た目とカーソルを追加する。

## データモデル

既存の `ClassNode.size` を利用する。

```javascript
{
  id: "class_1",
  position: { x: 120, y: 80 },
  size: {
    width: 230,
    height: 150
  }
}
```

保存される `size.height` はユーザーが指定した高さを保持する。SVG 上の実表示高さは以下で決める。

```text
renderWidth  = max(MIN_CLASS_WIDTH, classNode.size.width)
renderHeight = max(MIN_CLASS_HEIGHT, classNode.size.height, measureClassContentHeight(classNode))
```

この方針により、メンバーが多いクラスを小さくしようとしてもテキスト表示が潰れない。一方で、メンバーを減らした場合は保存済み高さに戻るため、ユーザーが意図して作った余白は保持される。

## 主要設計詳細

### リサイズ方向

初期実装では以下の3方向を扱う。

- `east`: 右端。幅のみ変更する。
- `south`: 下端。高さのみ変更する。
- `south-east`: 右下角。幅と高さを変更する。

### 最小サイズ

`src/models/diagramModel.js` と `src/views/svgRenderer.js` の両方で同じ値を使う必要がある。グローバルスクリプト構成のため、まず `diagramModel.js` に以下を定義し、後続で読み込まれる `svgRenderer.js` から参照する。

```javascript
const CLASS_MIN_SIZE = {
  width: 180,
  height: 110
};
```

既存の幅 `220` より少し小さくできるようにしつつ、クラス図として最低限読める幅を確保する。

### 表示サイズ計算

`SvgRenderer` に `getClassRenderSize(classNode)` を追加し、次の箇所から使う。

- `renderClassNode()`
- `relationshipPath()`
- `centerOf()`
- `edgePoint()`
- `classBounds()`

既存の `centerOf()`、`edgePoint()`、`classBounds()` はファイル下部の純関数だが、表示サイズ計算に `measureClassHeight()` が必要になるため、`SvgRenderer` のメソッドへ移すか、サイズを引数として受け取る形に変更する。変更量を抑えるため、`relationshipPath()` と範囲選択側でサイズを渡す形にする。

## リスクと対策

- リスク: リサイズハンドルの `pointerdown` がクラス移動の `pointerdown` と競合する。
  - 対策: ハンドル側で `event.stopPropagation()` と `setPointerCapture()` を実行し、`drag.type = "resize"` を明示する。
- リスク: 表示高さと保存高さがずれて接続線がボックスの内側に刺さる。
  - 対策: 接続線計算も `getClassRenderSize()` の結果を使う。
- リスク: リサイズ中に LocalStorage 書き込みが連発する。
  - 対策: `onClassResize` では `shouldSave=false`、`onClassResizeEnd` でのみ保存する。
- リスク: 古い保存データや Mermaid インポートデータに `size` がない。
  - 対策: `DiagramModel.normalizeState()` と `createClass()` で `normalizeClassSize()` を通す。
- リスク: 内容より小さい高さにした場合にテキストがはみ出す。
  - 対策: 実表示高さを `max(savedHeight, measuredContentHeight)` にする。

## 検証方針

- `index.html` をブラウザで開き、初期クラスを選択してハンドルが表示されることを確認する。
- 右端ドラッグで幅のみ変わることを確認する。
- 下端ドラッグで高さのみ変わることを確認する。
- 右下角ドラッグで幅と高さが変わることを確認する。
- 最小サイズ以下へドラッグしても、幅180px・高さ110px未満にならないことを確認する。
- 関係線がある2クラスの片方をリサイズし、線の端点が外周へ追従することを確認する。
- リサイズ後にページを再読み込みし、サイズが復元されることを確認する。
- クラス移動、範囲選択、Alt/中ボタンパン、Ctrl+ホイールズーム、関係選択が従来通り動くことを確認する。
- Mermaid エクスポートにサイズ情報が出力されないことを確認する。
- Mermaid インポート後のクラスをリサイズできることを確認する。
