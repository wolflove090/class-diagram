# クラスサイズ可変 要件書

## 背景と目的

現在のクラス図エディタでは、クラスノードの内部データに `size` が存在する一方で、SVG 描画時の高さはプロパティ数・メソッド数から再計算され、ユーザーがクラスボックスの表示サイズを直接変更する操作は提供されていない。

クラス名、プロパティ、メソッドの長さや図全体の密度に応じて、ユーザーがクラスボックスの幅と高さを調整できるようにする。これにより、長いメンバー名の見切れを避けたり、図上の余白を調整したりできるようにする。

## スコープ内

- SVG 上のクラスボックスにリサイズ用ハンドルを表示する。
- ユーザーがクラスボックスの端または角をドラッグしてサイズを変更できる。
- リサイズ中にクラスボックスの表示サイズを即時反映する。
- リサイズに合わせて接続線の始点・終点を更新する。
- リサイズ後のサイズを内部データモデルの `classNode.size` に保存する。
- LocalStorage 自動保存により、再読み込み後も変更後のサイズを復元する。
- Mermaid インポート時や既存保存データ復元時に、サイズ情報がないクラスには既定サイズを設定する。
- クラス内容が読めなくなるほど小さくならないよう、最小幅・最小高さを設ける。
- 既存のクラス移動、範囲選択、キャンバスパン、ズーム、関係選択の操作と衝突しないようにする。

## スコープ外

- Mermaid 記法へのサイズ情報のエクスポート。
- Mermaid 記法からのサイズ情報のインポート。
- 複数クラスを同時にリサイズする操作。
- キーボードによるリサイズ操作。
- リサイズ時の自動レイアウト再計算。
- クラス内テキストの折り返し、フォントサイズ変更、メンバー表示省略。
- インスペクター上で数値入力により幅・高さを変更する機能。

## 機能要件

### 1. リサイズハンドル表示

- クラスノードが選択されている場合、クラスボックスの外周にリサイズハンドルを表示すること。
- ハンドルはドラッグ可能であることが視覚的に分かる形にすること。
- 複数クラス選択時は、個別クラスのリサイズハンドルを表示しないこと。
- 接続モードで接続元として選択されている表示と、リサイズハンドル表示が同時に崩れないこと。

### 2. ドラッグによるサイズ変更

- ユーザーはクラスボックスの端または角をドラッグして、幅または高さ、またはその両方を変更できること。
- リサイズ開始時は、クラス移動ドラッグとして扱わないこと。
- リサイズ中は、ポインター位置に応じて対象クラスの `size.width` と `size.height` を更新すること。
- リサイズ中は、接続線の端点も変更後のクラスサイズに追従すること。
- リサイズ終了時に選択状態を維持すること。
- クリックだけでハンドルを押して離した場合、クラス選択状態を維持し、不要なサイズ変更を保存しないこと。

### 3. 最小サイズ制約

- 幅は既存表示の既定値を基準に、少なくともクラス名と基本余白を表示できる最小値を下回らないこと。
- 高さはプロパティ欄・メソッド欄を最低限表示できる最小値を下回らないこと。
- プロパティ数・メソッド数が多い場合、内容に必要な高さを下回っても表示が破綻しないよう、実表示高さは「保存済み高さ」と「内容に必要な高さ」の大きい方を使うこと。
- 保存する `size.height` はユーザー操作で指定した値を保持し、内容増減後の再描画で不必要に失われないこと。

### 4. データ保存と復元

- リサイズ後のサイズは `DiagramModel.updateClass()` を通してモデルに反映すること。
- リサイズ完了時に LocalStorage へ保存すること。
- リサイズ中の一時的な描画更新では、過剰な LocalStorage 書き込みを避けること。
- 古い保存データに `size` がない場合でも、`normalizeState()` により既定サイズを補完すること。

### 5. 既存操作との整合性

- クラス本体をドラッグした場合は、従来通りクラス移動として扱うこと。
- キャンバス背景をドラッグした場合は、従来通り範囲選択として扱うこと。
- Alt キーまたは中ボタンによるパン操作は、従来通り動作すること。
- Ctrl + ホイールによるズーム操作は、従来通り動作すること。
- リサイズハンドル上のポインターイベントは、クラス本体の `pointerdown` と競合しないこと。

## 非機能要件

- プレーンな JavaScript と SVG の既存構成を維持し、新しい外部ライブラリを追加しないこと。
- 既存の `DiagramModel`、`SvgRenderer`、`DiagramController` の責務分離を維持すること。
- リサイズ中の再描画が体感上不自然に遅くならないこと。
- クラス数が50、関係数が100程度の図でも、リサイズ操作が実用的に動作すること。
- リサイズ機能追加により、既存のインポート、エクスポート、クラス移動、関係作成、関係削除が退行しないこと。
- SVG 要素のカーソル、選択枠、ハンドルは視認性を保ち、主要なテキストを覆いすぎないこと。

## 受け入れ条件

- クラスを選択すると、クラスボックスの外周にリサイズハンドルが表示される。
- クラスボックス右端をドラッグすると、幅を拡大・縮小できる。
- クラスボックス下端をドラッグすると、高さを拡大・縮小できる。
- クラスボックス右下角をドラッグすると、幅と高さを同時に拡大・縮小できる。
- 最小サイズより小さくドラッグしても、クラスボックスは最小サイズ未満にならない。
- リサイズ中に接続線がクラスボックスの新しい外周へ追従する。
- リサイズ完了後にページを再読み込みしても、変更後のサイズが復元される。
- リサイズ操作後もクラス移動、範囲選択、パン、ズーム、関係選択が従来通り動作する。
- Mermaid エクスポート結果にはサイズ情報が含まれない。
- Mermaid インポートしたクラスは既定サイズで表示され、その後リサイズできる。

## 未確定事項・確認事項

- ハンドルの数は、初期実装では右端、下端、右下角の3種類を想定する。左端・上端・四隅すべての対応が必要な場合は実装範囲を拡張する。
- 高さを内容より小さくした場合に、内容をクリップするか、常に内容に必要な高さまで広げるかは設計で確定する。現時点では、表示破綻を避けるため「実表示高さは内容に必要な高さ以上」とする案を採用候補とする。
- リサイズハンドルを選択中のみ表示するか、ホバー時にも表示するかは設計で確定する。現時点では、誤操作を減らすため「選択中のみ表示」を採用候補とする。
