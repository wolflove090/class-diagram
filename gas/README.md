# GASアクセスKPI API

`Code.gs`は、設計メーカーのユニーク表示回数をGoogleスプレッドシートへ日別集計するGAS Webアプリです。

## 1. 集計先スプレッドシートを作成する

Googleスプレッドシートを新規作成し、URL中の`/d/`と`/edit`の間にあるIDをコピーします。

## 2. GASプロジェクトへコードを登録する

スプレッドシートで **拡張機能 → Apps Script** を開き、既存のコードを`Code.gs`の内容へ置き換えます。先頭の`SPREADSHEET_ID`へ、手順1でコピーしたIDを設定して保存します。

プロジェクトの設定でタイムゾーンを **(GMT+09:00) 日本標準時 - 東京** にしておくと、管理画面で日時を扱う際にも混乱しません。APIはコード内でJSTを明示しているため、集計日自体もJSTになります。

## 3. Webアプリとしてデプロイする

1. **デプロイ → 新しいデプロイ** を選びます。
2. 種類の選択で **ウェブアプリ** を選びます。
3. 「次のユーザーとして実行」は **自分** を選びます。
4. 「アクセスできるユーザー」は、GitHub Pagesから呼べる公開設定（通常は **全員**）を選びます。
5. デプロイ後に表示される`https://script.google.com/macros/s/.../exec`形式のURLをコピーします。

コードを変更した場合は、新しいバージョンを作成してデプロイを更新してください。`/dev` URLは開発者向けのため、GitHub Pagesには`/exec` URLを設定します。

## 4. GitHub PagesへURLを設定する

`src/main.js`の`endpointUrl`に、手順3の`/exec` URLを貼り付けます。

```js
const analytics = new AnalyticsService({
  endpointUrl: "https://script.google.com/macros/s/ここにデプロイID/exec"
});
```

## 保存されるデータ

- `DailyKPI`：日付、表示回数
- `VisitorLog`（自動で非表示）：`日付:匿名IDのSHA-256ハッシュ`

同じブラウザは`localStorage`の匿名IDにより、JSTで1日1回だけカウントされます。ブラウザのストレージを削除した場合、別のブラウザや端末で開いた場合は別の利用者として扱われます。

公開WebアプリのURLはブラウザから確認できるため、悪意ある送信を完全に防ぐ仕組みではありません。用途は厳密な監査ではなく、利用傾向のKPI把握を想定しています。
